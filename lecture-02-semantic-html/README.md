## 1. What I implemented this lecture

- Structured the page with header, main content sections, and footer
- Added a portfolio section with images, links, and an embedded video
- Included an areas of interest section using lists
- Applied basic accessibility features

---

## 2. Semantic decisions I made (REQUIRED)

### Decision 1
- **Element(s) used:** `<header>`
- **Where in the page:** At the top of the page
- **Why this element is semantically correct:**  
  The `<header>` element contains introductory information such as my name and a short description, which clearly identifies the page content.

### Decision 2
- **Element(s) used:** `<section>` and `<article>`
- **Where in the page:** Portfolio / Works section
- **Why this element is semantically correct:**  
  `<section>` groups related content, and `<article>` represents a standalone piece of content describing a project.

### Decision 3
- **Element(s) used:** `<figure>` and `<figcaption>`
- **Where in the page:** Around images and the embedded video
- **Why this element is semantically correct:**  
  These elements associate media with descriptive text, improving structure and accessibility.

## 3. Accessibility considerations
- Added a skip link to allow users to jump directly to the main content
- Used a correct heading hierarchy (h1 → h2 → h3)
- Added alt text for all images
- Used semantic HTML elements instead of generic divs
